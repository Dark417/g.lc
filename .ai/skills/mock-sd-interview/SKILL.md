# mock-sd-interview

1. Pick level, role, and question.
2. Start with clarifying questions.
3. Drive through requirements, architecture, deep dives, and tradeoffs.
4. Interrupt with realistic bottleneck/failure/cost/security follow-ups.
5. Score performance and output next-step prep plan.
