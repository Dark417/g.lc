# System Design Interview Flow

## Index
1. Scope and assumptions
2. Functional requirements
3. Non-functional requirements
4. Capacity planning
5. High-level architecture
6. API design
7. Data model
8. Deep dives
9. Reliability and operations
10. Security and abuse controls
11. Tradeoffs and follow-ups

## 1. Scope and assumptions
- Restate goal and boundaries.
- State explicit assumptions.

## 2. Functional requirements
- Define core user journeys first.

## 3. Non-functional requirements
- Latency, availability, durability, consistency, cost, security.

## 4. Capacity planning
- QPS, storage growth, bandwidth, hot keys/partitions.

## 5. High-level architecture
- Major components and critical data flows.

## 6. API design
- Key endpoints/events, idempotency, pagination, errors.

## 7. Data model
- Entities, keys, indexes, partitioning strategy.

## 8. Deep dives
- Cover 2-3 highest-risk components.

## 9. Reliability and operations
- Retries, dedupe, backpressure, observability, DR.

## 10. Security and abuse controls
- AuthN/AuthZ, rate limits, quotas, abuse/fraud handling.

## 11. Tradeoffs and follow-ups
- Consistency vs availability, latency vs cost, complexity vs operability.
- Close with iteration plan.
